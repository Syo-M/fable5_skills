# v1.9.0 trigger evaluation — consolidated summary (2026-07-02)

First measured answer to "do the rules actually FIRE?" — 26 golden prompts (JP/EN) run headless
in a fresh Vite+React fixture project, activations observed by hooks (see ../README.md for method).
Three measurement series, each single-run (stochastic — treat per-prompt rates as signal, not truth):

| Series | Scope | Valid? | Result |
|---|---|---|---|
| 1. Baseline (v1.8.0 rules) | all 26 | ✅ | **12/26** |
| 2. Post-fix re-run | the 14 baseline failures | ✅ | **9/14 flipped to PASS** |
| 3. Post-fix full run | all 26 | ⚠️ prompts 1–15 only (env failure after — see in-file caveat) | 11/15 valid-PASS, corroborates series 2 |

## The finding that mattered

Baseline failures were NOT missing trigger anchors (the failing phrasings already sat in the
descriptions). The model simply **skipped loading "rulebook" skills for tasks it could do
directly** (styling, charts, stories, unit tests) — while agents (5/5), path-scoped rules, and
procedure skills fired reliably.

**Fix**: CLAUDE.md now states "Loading skills is not optional — find the matching row and LOAD
that skill before writing code." Effect: 9/14 previous failures flipped immediately; series 3
additionally flipped `story-jp`.

## Consolidated post-fix state

- **22/26 prompts have ≥1 observed PASS** across valid post-fix measurements.
- **4 persistent failures** (never passed in any valid run): `security-form-jp`, `chart-jp`,
  `chart-en`, `react-state-jp`. All have matching anchors; these are residual stochastic skips on
  "just do the work" prompts. Mitigation: the layered design held anyway — in 3 of 4 cases a path
  rule or the CLAUDE.md floor carried the relevant invariants; tracked for the next description
  tuning round (see eval/README "Interpreting failures").
- Layer reliability ranking (consistent across all series):
  **agents > path rules > procedure skills > domain-rulebook skills**.

## Operational lessons (encoded in docs/code)

1. A single hook event name unknown to the INSTALLED CLI silently disables ALL hooks in
   settings.json (verified on 2.1.87). → warning in README + run-eval.mjs; always fire-test.
2. Long headless series can hit usage limits and return silent zero-activity results — a run
   where previously-bulletproof prompts suddenly observe NOTHING is a broken harness, not a
   regression. → run3 tail invalidated on this basis.
3. Single-run rates are noisy; `--runs 3` is the release-grade protocol (MAINTENANCE.md).
